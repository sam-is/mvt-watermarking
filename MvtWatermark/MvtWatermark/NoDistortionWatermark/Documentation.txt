	При встраивании ЦВЗ, если одна из подпоследовательностей будет состоять целиком из False, то есть при переводе в десятичную систему счисления 
получится 0, NonStaticMapboxTileWriterWm сгенерирует исключение.

	При извлечении ЦВЗ, в изначально созданном BitArray (размера: [количество тайлов в дереве] * [options.Nb]), состоящем исключительно из False,
с нужного индекса перезаписываются подпоследовательности этого битового массива, заменяясь на фрагменты ЦВЗ, извлеченные из тайлов.
Эта перезапись происходит только если из тайла удалось извлечь фрагмент ЦВЗ. Если извлечение не удалось, ничего не перезаписывается, 
и продолжается извлечение уже из следующего тайла.
Соответственно, если из одного или нескольких тайлов не произошло извлечение, то в конце результата-массива BitArray остаётся подпоследовательность, 
состоящая исключительно из False, и имеющая размер: [количество тайлов, из которых не получилось извлечь фрагмент ЦВЗ] * [options.Nb].

С полигонами не работаем по причине того, что полигон представляет собой (можно сказать) один лайнстринг (не мультилайнстринг), который
закрывается командой ClosePath. Этот алгоритм же любой лайнстринг превращает в мультилайнстринг, соответственно команда ClosePath будет
после встраивания делать полигон только из последнего лайнстринга, визуально изменяя полигон. Можно тогда вообще не пользоваться полигоном,
превратив его чисто в мультилайнстринг, закрывая его руками - то есть дополнительным сегментом, но как тогда быть со стилями, например?
